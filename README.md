## 💜 Sobre mim

Sou estudante de **Ciência da Computação na UniSantos** (formatura prevista para dezembro de 2027), com formação técnica em programação pela **Escola e Faculdade Fortec** e certificação **AWS Certified Cloud Practitioner**.

Trabalho no desenvolvimento full stack — do front-end em React/TypeScript ao back-end em Node.js/Express com bancos SQL e NoSQL — e venho me aprofundando em **ciência de dados e machine learning**, aplicando esse conhecimento em projetos práticos com dados reais.

Gosto de entender o sistema de ponta a ponta: modelagem de dados, arquitetura da aplicação, e a experiência final do usuário.

## 🚀 Projetos em Destaque

<details>
<summary><b>🍕 Sistema de Pizzaria — Mama Mia</b></summary>
<br/>

Sistema web para gerenciamento de pizzaria, com cardápio interativo, carrinho de compras e painel administrativo. Desenvolvido em grupo aplicando conceitos de Programação Orientada a Objetos (herança, polimorfismo, abstração e encapsulamento).

| | |
|---|---|
| **Stack** | Python 3.x, Flask, HTML/CSS/JavaScript |
| **Funcionalidades** | Personalização de pizzas (tamanho, ingredientes, borda recheada), checkout com múltiplas formas de pagamento (cartão, PIX, dinheiro), painel admin com gerenciamento de pedidos |
| **Arquitetura** | Classes dedicadas (`Pizza`, `Pedido`, `Usuario`/`Cliente`/`Administrador`, `Pizzaria`) |
| **Repositório** | [Gustavo-Mart/Pizzaria](https://github.com/Gustavo-Mart/Pizzaria) |

Projeto acadêmico desenvolvido em equipe (3 integrantes), com foco em fixar princípios de POO em um sistema completo, do cardápio ao checkout.

</details>

<details>
<summary><b>🎵 MusicXP — The Widget for Music</b></summary>
<br/>

Widget de música para desktop com interface inspirada na estética do Windows XP, empacotado como aplicativo Electron para Windows e Linux.

| | |
|---|---|
| **Stack** | JavaScript, Electron, Tailwind CSS |
| **Plataformas** | Windows (`.exe`) e Linux (`.deb`) |
| **Repositório** | [Gustavo-Mart/MusicXP-Website](https://github.com/Gustavo-Mart/MusicXP-Website) |

Projeto pessoal explorando empacotamento de aplicações desktop com Electron e uma identidade visual retrô nostálgica.

</details>

<details>
<summary><b>📝 Blog-Mongo</b></summary>
<br/>

Aplicação de blog full stack (MERN) com armazenamento de imagens em base64 diretamente no MongoDB — solução adotada após abandonar o Cloudinary por conflitos de tipagem em TypeScript.

| | |
|---|---|
| **Stack** | React, TypeScript, Node.js, Express, MongoDB/Mongoose |
| **Destaque técnico** | Armazenamento de imagens em base64 no MongoDB, decisão pragmática diante de conflitos de tipagem com serviços externos de imagem |
| **Repositório** | [Gustavo-Mart/Blog-Mongo](https://github.com/Gustavo-Mart/Blog-Mongo) |

</details>

<details>
<summary><b>🚧 Modelo-PRF — Análise Preditiva de Acidentes Rodoviários</b></summary>
<br/>

Pipeline completo de dados e machine learning sobre acidentes em rodovias federais brasileiras (dados da PRF, 2021–2025): normalização de CSVs em SQLite, scripts em Python (Google Colab), mapas interativos com Folium e um classificador Random Forest para prever acidentes fatais vs. não fatais.

| | |
|---|---|
| **Stack** | Python, SQLite, Pandas, Scikit-learn, Folium |
| **Técnicas de ML** | `RandomizedSearchCV` para tuning, `RandomUnderSampler` para desbalanceamento de classes, `MinMaxScaler` para normalização, ajuste de threshold e F1-score como métrica |
| **Repositório** | [Gustavo-Mart/Modelo-PRF](https://github.com/Gustavo-Mart/Modelo-PRF) |

Projeto de estudo aplicado, cobrindo o pipeline de dados de ponta a ponta: da limpeza dos dados brutos até a avaliação de um modelo classificador.

</details>
